Small-screen hero meta

Compare the small-screen hero meta layouts

The homepage now keeps the original inline row through medium widths and switches to the cards layout only on narrower phones (520px and below). These previews stay here as a quick way to compare the three mobile fallback treatments on the real page.

Homepage
Option 1

Stacked pills

Two full-width rows with labels above the values. Safest and most readable.

Best if we want the hero header to feel calm and explicit, even if it gets a little taller.

Option 2

Mini stat cards

Splits the row into two compact cards so the date and count read like dashboard stats.

Best if we want the metadata to feel more intentional and data-forward. This is the current default for very small screens.

Option 3

Compact chips

Turns both items into short pills that wrap cleanly without adding much height.

Best if we want the smallest footprint and the least visual interruption in the hero.
