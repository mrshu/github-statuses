About this mirror

GitHub stopped updating its status page with aggregate uptime numbers some time ago — if you use it regularly, you might have a feeling why. This is the missing version.

We rebuild platform‑wide and per‑service uptime from archived status updates, derive minute‑level downtime windows, and map incidents to services whenever the source data allows. Everything is open source, and PRs are very welcome!

This was also built as a sanity-keeping exercise with the help of a bunch of agents by Marek Šuppa (@mareksuppa). The code lives on GitHub.

The header repo shortcut was suggested by pRizz in issue #3.

Nerd notes: the pipeline reconstructs history by replaying the Atom feed snapshots committed to git via Flat Data. It rebuilds each incident timeline and computes uptime using merged downtime windows rather than day‑level buckets. When components are missing, we infer tags with GLiNER2 but only keep labels that are explicitly grounded in the incident text.
